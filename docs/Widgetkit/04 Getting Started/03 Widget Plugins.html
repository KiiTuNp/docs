Widget Plugins

A widget plugin determines how your widget will be displayed in the frontend, for example as a Slideshow or a Grid. You can also add custom widget plugins. Each plugin accepts a set of configuration parameters. These can be set using the visual interface provided by Widgetkit or by changing the shortcode directly.

Accordion

The Accordion widget display content inside a list of items that can be expanded and collapsed by clicking each of their headers.

Section Description
Accordion Determine whether or not multiple items can be opened at the same time.
Media Determine, if and how media and overlay of all items are rendered. You can apply different animations to the overlay and the images.
NOTE The link as well as the Social Buttons can be added in the {doc:content-manager text:Content Manager} and through {doc:manage-fields text:Custom fields} when creating the content.
Content Determine, if and how the content of all items is rendered.
NOTE The link as well as the Social Buttons can be added in the {doc:content-manager text:Content Manager} and through {doc:manage-fields text:Custom fields} when creating the content.
General Set a custom HTML class that will be added to the widget element in the rendered output.

Example Shortcode

This is a default Accordion widget. An open item will collapse automatically when another one expands.

[widgetkit id="23" widget="accordion"]

An accordion widget that allows multiple opened items and links the media overlay to the URL you have set in the {doc:content-manager text:Content Manager}.

[widgetkit id="23" widget="accordion" collapse=0 media_overlay="link"]

Gallery

The gallery widget displays images and videos in a grid, automatically creating thumbnails for your media items. Use the widget configuration to change the appearance of the gallery.

Section Description
Layout Configure the behavior of the gallery grid, the number of columns for different screen sizes (for responsive behaviour) and add animations to the gallery items.
Media Determine, if and how media and overlay of all items are rendered. You can apply different animations to the overlay and the images.
Content Determine, if and how the content of all items is rendered.
NOTE The link as well as the Social Buttons can be added in the {doc:content-manager text:Content Manager} and through {doc:manage-fields text:Custom fields} when creating the content.
Lightbox Enable a lighbox to display your media fully sized or set a fixed width and height. You can use the title or the content field as a text caption.
General Set a custom HTML class that will be added to the widget element in the rendered output.

Example Shortcode

Default gallery widget. Content items will have 100% each and will therefore be stacked. The content is displayed in an image caption and the overlay content is an icon.

[widgetkit id="23" widget="gallery"]

A gallery widget that displays 4 columns on large screens automatically collapses to 2 columns on tablets and such and displays a single column on all devices smaller than that. To understand this mobile first approach, have a look at the UIkit responsive grid that is used here. The overlay is centered and title, content as well as the lightbox link, which are displayed as small icon, are located inside the overlay.

[widgetkit id="23" widget="gallery" columns_medium="2" columns_large="4" overlay="center" lightbox_link=1 lightbox_style="icon-small"]

Grid

In a Grid widget, all items will be displayed as single items of a grid. The height of grid items can be matched or displayed in a dynamic grid, always applying the same spacing between items regardless of their height.

Section Description
Layout Configure the behavior of the grid, number of columns for different screen sizes (for responsive behaviour) and set the panel appearance. The gutter property determines the spacing between columns and rows. To link the entire panel, you can add a link in the {doc:content-manager text:Content Manager}.
Media Determine, if and how media and overlay of all items are rendered. You can apply different animations to the overlay and the images.
NOTE The link as well as the Social Buttons can be added in the {doc:content-manager text:Content Manager} and through {doc:manage-fields text:Custom fields} when creating the content.
Content Determine, if and how the content of all items is rendered.
NOTE The link as well as the Social Buttons can be added in the {doc:content-manager text:Content Manager} and through {doc:manage-fields text:Custom fields} when creating the content.
General Set a custom HTML class that will be added to the widget element in the rendered output.

Example Shortcode

Default grid widget. Content items will have 100% each and will therefore be stacked.

[widgetkit id="23" widget="grid"]

A dynamic grid widget that displays 4 columns on large screens automatically collapses to 2 columns on tablets and such and displays a single column on all devices smaller than that. To understand this mobile first approach, have a look at the UIkit responsive grid that is used here.

[widgetkit id="23" widget="grid" grid="dynamic" columns_medium="2" columns_large="4"]

Grid Slider

A Grid Slider widget allows you to display multiple Slideshows within one grid. Mind that when creating content for this widget, you need to add another media field for every slide your slideshow is going to have. Each slide will be displayed with the same content and title field within one item.

Section Description
Layout Configure the behavior of the grid, number of columns for different screen sizes (for responsive behaviour) and set the panel appearance. The gutter property determines the spacing between columns and rows. To link the entire panel, you can add a link in the {doc:content-manager text:Content Manager}.
Media Configure the dimensions, alignment and general behavior of the media items.
Slideshow Configure the appearance and behaviour of the slideshow. Settings for the navigation and animations can be found here as well.
Content Determine, if and how the content of all items is rendered.
NOTE The link as well as the Social Buttons can be added in the {doc:content-manager text:Content Manager} and through {doc:manage-fields text:Custom fields} when creating the content.
General Set a custom HTML class that will be added to the widget element in the rendered output.

Example Shortcode

Default grid widget. Content items will have 100% each and will therefore be stacked.

[widgetkit id="23" widget="grid"]

A dynamic grid widget that displays 4 columns on large screens automatically collapses to 2 columns on tablets and such and displays a single column on all devices smaller than that. To understand this mobile first approach, have a look at the UIkit responsive grid that is used here.

[widgetkit id="23" widget="grid" grid="dynamic" columns_medium="2" columns_large="4"]

Grid Stack

In a Grid stack, all items from your selected content will be rendered below each other. This can easily be used for a paired display of an image (taken from the media field of your content item) and a short text (taken from the content field of your content item).

Section Description
Layout Configure the appearance of the grid, for example the relation of content and media. The grid gutter determines the spacing between items in the grid. You can individually set animations for the media and content columns.
Media Determine, if and how media and overlay of all items are rendered. You can apply different animations to the overlay and the images.
NOTE The link as well as the Social Buttons can be added in the {doc:content-manager text:Content Manager} and through {doc:manage-fields text:Custom fields} when creating the content.
Content Determine, if and how the content of all items is rendered.
NOTE The link as well as the Social Buttons can be added in the {doc:content-manager text:Content Manager} and through {doc:manage-fields text:Custom fields} when creating the content.
General Set a custom HTML class that will be added to the widget element in the rendered output.

Example Shortcode

Default grid stack widget.

[widgetkit id="23" widget="grid-stack"]

A grid stack widget with dividers between the items, the media elements cropped to circles and are animated to slide in when the item scrolls into view.

[widgetkit id="23" widget="grid-stack" divider=1 animation_media="slide" animation_content="slide" media_border="circle"]

Map

A map widget displays a Google map on your website. Make sure to add a custom field Location to your content element and select a location you want to display in the map. The other fields of your content will be displayed when you click on the location marker (provided the Popup is enabled). You can add several pins and popup boxes by adding more items to your content element. For more information, read the {doc:manage-fields text:Manage fields} doc.

Section Description
Map Configure the appearance and behaviour of the map and enable the Popup that will display media and content of each item.
Style Configure the styling of the map, for example invert the colours or set custom colours.
Media Determine, if and how media and overlay of all items are rendered. You can apply different animations to the overlay and the images.
NOTE The link as well as the Social Buttons can be added in the {doc:content-manager text:Content Manager} and through {doc:manage-fields text:Custom fields} when creating the content.
Content Determine, if and how the content of all items is rendered.
NOTE The link as well as the Social Buttons can be added in the {doc:content-manager text:Content Manager} and through {doc:manage-fields text:Custom fields} when creating the content.
General Set a custom HTML class that will be added to the widget element in the rendered output.

Example Shortcode

A simple map widget. Make sure the content (#23 in the example) has a custom field added for the location.

[widgetkit id="23" widget="map"]

A map showing the satellite image view and without any controlling UI elements.

[widgetkit id="23" widget="map" maptypeid="SATELLITE" disabledefaultui=1]

Slideshow

A slideshow turns your selected content element into a sequence of slides. Each item in your content element will represent one slide. Select a media element as the slide's background. The title and content fields can be displayed in an overlay on top of that image.

Section Description
Slideshow Configure the appearance and behaviour of the slideshow. Settings for the navigation and animations can be found here as well as the option to enable a fullscreen slideshow.
Media Here you you can set the image proportions and enable an overlay. The overlay content is taken from the title and content fields in your content item.
Content Determine, if and how the content of all items is rendered.
NOTE The link as well as the Social Buttons can be added in the {doc:content-manager text:Content Manager} and through {doc:manage-fields text:Custom fields} when creating the content.
General Set a custom HTML class that will be added to the widget element in the rendered output.

Example Shortcode

The default slideshow.

[widgetkit id="23" widget="slideshow"]

A slideshow with captions enabled and animations for slides and a centered overlay.

[widgetkit id="23" widget="slideshow" animation="swipe" overlay=1 overlay_position="center"]

Switcher

A switcher consists of a number of navigation toggles and their related content items. Switchers are useful for realizing little tabbed navigations and similar. Each item in your selected content element will result in a navigation entry.

Section Description
Layout Configure the appearance and behavior of the switcher. The navigation can be of different visual appearance and can be positioned at any side of the content.
Media Determine, if and how media and overlay of all items are rendered. You can apply different animations to the overlay and the images.
NOTE The link as well as the Social Buttons can be added in the {doc:content-manager text:Content Manager} and through {doc:manage-fields text:Custom fields} when creating the content.
Content Determine, if and how the content of all items is rendered.
NOTE The link as well as the Social Buttons can be added in the {doc:content-manager text:Content Manager} and through {doc:manage-fields text:Custom fields} when creating the content.
General Set a custom HTML class that will be added to the widget element in the rendered output.

Example Shortcode

The default switcher.

[widgetkit id="23" widget="switcher"]

A switcher with a tabbed navigation below the content.

[widgetkit id="23" widget="switcher" nav="tabs" position="bottom"]

A switcher with thumbnails for a navigation, which is centered.

[widgetkit id="23" widget="switcher" nav="thumbnails" alignment="center"]
